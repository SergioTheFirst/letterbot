# ADR-004: Deadlock → Meeting Moment (shadow)

## Status
Accepted (shadow-only)

## Context
Длинные треды и задержки могут означать deadlock в коммуникации.

## Decision
- Детектировать DEADLOCK_DETECTED из цепочек событий.
- В v1.5 только запись события.

## Data prerequisites (blocked)
- В текущих таблицах нет стабильного thread_id / conversation_id.
- Требуется надёжный thread key: thread_id / conversation_id или устойчивый in_reply_to/references mapping.

## Consequences
- Будет основа для будущего "meeting moment" предложения.
