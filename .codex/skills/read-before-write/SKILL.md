# read-before-write

Use this skill before any repo edits.

## Required steps
1. Read `AGENTS.md`, `CONSTITUTION.md`, and `CONTINUITY.md` (if missing, create it in required format).
2. Post READ RECEIPTS listing the files read and the working-set files you will modify.
3. Make only minimal, scoped edits aligned with current `Goal/Now/Next`.
4. Run at least one verification command (for example `python -m compileall mailbot_v26` or a targeted `pytest` command) before finishing.

## Enforcement
- Do not edit files before step 1 and step 2 are complete.
- If instructions conflict, stop and escalate with explicit alternatives.
